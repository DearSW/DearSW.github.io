Hexo项目创建记录

  1. npm install -g hexo-cli 全局安装hexo命令行。

  2. hexo init <项目名> 创建项目，可以是任意的项目名。

  3. cd <项目名> 切换到项目所在目录。

  4. npm install 自动安装项目依赖。

Hexo相关命令记录

  1. hexo n <文章名> 创建一个新文章md文件，具体放置位置，上面的图中已说明。

    1. 注意：图片写法差异如下图

  2. hexo g 文章写完保存之后，使用这个命令来生成。注意：添加了任何文件都要使用这个命令去生成，然后才能部署。

  3. hexo d 文章生成完毕之后，再使用这个命令把项目部署到github网站上，当然，在部署之前是需要创建一个github仓库的，而且这个仓库的名字是固定的，github官方规定为 github账户名.github.io，即每个用户只能拥有一个这样的仓库。

    1. 如下图

  4. hexo clean 有时修改了fexo主题的配置文件，部署之后发现没有生效，那么可以执行这个命令，那么就会重新生成相关文件，之后再部署就会生效了。
